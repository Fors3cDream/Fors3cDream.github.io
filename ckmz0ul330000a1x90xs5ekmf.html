Go并发

并发是独立活动的组合，比如Web服务器在同时处理多个用户请求，每个用户的请求是相互独立的。并发在现在很多程序中都存在。

Go有两种编写并发程序的方式。一是在通过线程实现的传统样式。二是通过称为goroutine的独立活动之间传递，并以进程进行通信。

在实现并发程序的最大问题是在进程之间共享数据。Go通过channel来回传递数据。这意味着只有一个活动(goroutine)有权访问数据，不存在竞争条件。Go中实现并发的方法是“不通过共享内存通信，而是通过通信共享内存”。

goroutine（轻量线程）

goroutine是轻量线程中的并发活动，而不是在操作系统中进行的传统活动。比如程序中实现一个输出函数和一个两数相加的函数，一个并发程序可以有多个goroutine同时调用这两个个函数。

如果要创建goroutine，则必须在调用该函数之前使用go关键字。

将channel作为通信机制

Go中的channel是goroutine之间的通信机制。这就是Go实现并发的方式-“不是通过共享内存通信，而是通过通信共享内存。”

当需要将值从一个goroutine发送到另一个goroutine时，将使用channel。

Channel语法

由于channel是发送和接收数据的通信机制，因此它也有类型之分。这意味着只能发送channel支持的数据类型。除了使用chan关键字作为channel数据类型之外，还需要指定通过channel传递的数据类型，比如int类型。

每次声明一个channel或希望在函数中指定一个channel作为参数时，都需要使用chan <type>，如chan int。要创建channel，需使用内置的make()函数，如下所示：

一个channel可以执行两项操作：发送数据和接收数据。若要指定channel具有的操作数据，需要使用channel运算符<-。在channel中发送数据和接收数据属于阻止操作。

如果希望channel仅发送数据，则必须在channel之后使用<-运算符。如果希望channel接收数据，则必须在channel之前使用<-运算符，如下所示：

~~Golang
ch <- x // 通过channel发送（或写入）数据
x = <- ch // x接收（或读取）发送到channel中的数据
<- ch // 接收数据，并丢弃

有缓冲channel

默认情况下channel是无缓冲行为。这意味着只有存在接收操作时，channel才接受发送操作。否则，程序将被永久阻止等待。

有时需要在goroutine之间进行此类同步。但是，有时可能只需要实现并发，而不需要限制goroutine之间的通信方式。

有缓冲channel在不阻止程序的情况下发送和接收数据，因为有缓冲channel的行为类似于队列。创建channel时，可以限制此队列的大小。

每次向channel发送数据时，都会将元素添加到队列中。然后，接收操作将从队列中删除该元素。当channel已满时，任何发送操作都将等待，直到有空间保存数据。相反，如果channel是空的且存在读取操作，程序则会被阻止，直到有数据可以读取。
